# 채용 공고 개발 매뉴얼

## 제품 정의

이 프로젝트는 공개 채용 서비스가 아니라 개인용 채용 모니터다.

목표는 공고 상세 본문을 복제하거나 요약하는 것이 아니라, 관심 있는 공공기관 기계직 채용공고가 올라왔다는 사실을 빠르게 감지하고 ALIO 상세 링크 또는 채용기관 원문 링크로 이동하게 만드는 것이다.

## 1차 개발 범위

이번 단계는 ALIO 공공데이터 API를 1순위로 쓰고, 허용된 채용 플랫폼 공개 목록을 보조 수집원으로 사용한다.

- 사기업/채용 플랫폼 크롤링은 robots.txt에서 허용되는 공개 목록 URL만 저빈도로 확인한다.
- 2차 상세 크롤링은 하지 않는다.
- 공고 제목, 기관명, 시작일, 종료일, 채용분야, 고용형태, 채용구분, 근무지, ALIO 상세 URL, 채용기관 외부 URL 정도만 저장한다.
- 채용 플랫폼은 누락 방지를 우선해 기본적으로 0점 이상 공고를 저장하고, 점수는 우선순위 표시용으로 사용한다.
- 신규 공고 감지와 점수 계산을 먼저 만든다.
- 웹 대시보드는 DB와 수집기가 안정화된 뒤 붙인다.

## 공식 참고

ALIO 오픈데이터 사이트는 메인 화면에서 `오픈API 찾기`, `오픈 API신청`, `채용정보` 메뉴를 제공한다.

- 메인: https://opendata.alio.go.kr/new/main.do
- 채용정보 조회: https://opendata.alio.go.kr/new/odaApiMng/recrutInquiryList.do
- 사용자 API 상세: https://opendata.alio.go.kr/new/odaApiUserInqDataMng/openApiRecrutDetail.do
- 공개 조회 endpoint: https://opendata.alio.go.kr/new/odaApiMng/recrutInquiryAjaxList.do
- API 활용신청은 로그인 후 진행된다.

조회 화면에서 확인한 1차 필터:

- 기관유형
- 기관분류
- 기관
- 채용분야: `기계` 포함
- 근무지
- 필요학력
- 고용형태
- 경력구분
- 시작기간/종료기간
- 대체인력 유무
- 상태: `진행`, `종료`

## 환경 설정

`.env.example`을 `.env`로 복사한 뒤 값을 채운다.

```text
ALIO_API_URL=ALIO 활용신청 후 확인한 채용정보 API URL
ALIO_API_KEY=발급받은 인증키
ALIO_API_KEY_PARAM=serviceKey
ALIO_DEFAULT_PARAMS={"pageNo":"1","numOfRows":"100","type":"json"}
ALIO_PUBLIC_URL=https://opendata.alio.go.kr/new/odaApiMng/recrutInquiryAjaxList.do
ALIO_PUBLIC_DEFAULT_PARAMS={"pageNo":"1","numOfRows":"100","ncsCdLst":"R600015","ongoingYn":"Y"}
MECH_JOB_RADAR_DB=data/job_radar.sqlite3
```

ALIO 응답 파라미터명은 활용신청 후 받은 명세를 기준으로 조정한다. 현재 코드는 JSON/XML 응답을 모두 받아보고, 흔한 필드명 후보를 느슨하게 매핑하도록 작성한다.

코드 문서 기준 NCS 기계 코드는 `R600015`다. 현재 코드는 응답 값이 `R600015`처럼 코드만 내려와도 `R600015 기계`로 확장해 점수 계산에 반영한다.

## 실행

```powershell
$env:PYTHONPATH='src'
python -m job_radar init-db
python -m job_radar check-config
python -m job_radar import-sample docs/samples/alio_recruitment_sample.json
python -m job_radar fetch-alio-public --dry-run --limit 5
python -m job_radar fetch-alio-public --limit 5
python -m job_radar fetch-platform --platform jobkorea --dry-run --limit 10
python -m job_radar fetch-platform --platform jobkorea --limit 10
python -m job_radar fetch-platform --platform saramin --dry-run --limit 10
python -m job_radar fetch-platform --platform saramin --limit 10
python -m job_radar fetch-all
python -m job_radar list-jobs --min-score 1
python -m job_radar logs
python -m job_radar write-dashboard
python -m job_radar serve
python -m job_radar fetch-alio --dry-run
python -m job_radar fetch-alio
```

`--dry-run`은 DB에 저장하지 않고 API 응답 매핑과 점수 계산만 확인한다.

`write-dashboard`는 `data/dashboard.html`을 생성한다. 별도 서버 없이 브라우저로 열어 전체 공고와 크롤링 로그를 확인할 수 있다. 저장된 모든 공고를 포함하고, 브라우저에서 검색어/출처/경력/점수 필터를 적용한다.

`serve`는 계속 실행되는 로컬 서버를 띄운다. 기본 주소는 `http://127.0.0.1:8787`이며, 시작 시 1회 수집하고 이후 KST 기준 00/06/12/18시에 자동 수집한다. 수동 갱신 버튼은 서버 모드에서만 표시된다.

대시보드의 기본 링크는 `recrutPblntSn` 기반 ALIO 상세 페이지다. 채용기관이 제공한 `srcUrl`은 회사 홈이나 채용 루트로 내려오는 경우가 있어 `외부 원문` 보조 링크로만 표시한다.

ALIO API에 추가 파라미터가 필요하면 다음처럼 넘긴다.

```powershell
python -m job_radar fetch-alio --param pageNo=1 --param numOfRows=100
```

공개 조회 endpoint에 추가 검색 파라미터를 넘길 수도 있다.

```powershell
python -m job_radar fetch-alio-public --param ongoingYn=Y --param ncsCdLst=R600015 --param numOfRows=100
```

채용 플랫폼 공개 목록을 확인할 때는 다음 명령을 쓴다.

```powershell
python -m job_radar fetch-platform --platform jobkorea --dry-run --limit 10
python -m job_radar fetch-platform --platform jobkorea --url "https://www.jobkorea.co.kr/recruit/joblist"
python -m job_radar fetch-platform --platform saramin --dry-run --limit 10
```

`fetch-platform`은 요청 전에 대상 URL의 `/robots.txt`를 확인한다. 차단된 경로, 로그인 후 접근 경로, 캡차나 차단 우회가 필요한 경로는 수집하지 않는다. 기본 구현은 목록 페이지에서 제목, 회사명, 링크만 저장하고 상세 본문은 복제하지 않는다.
기본 저장 기준은 `--min-score 0`이다. 더 좁게 보고 싶을 때만 `--min-score 8`처럼 올린다.

## ALIO 활용신청 후 체크리스트

1. https://opendata.alio.go.kr/new/main.do 에 로그인한다.
2. `오픈 API신청`에서 채용정보 API를 신청한다.
3. 명세에서 다음 값을 확인한다.
   - 호출 URL
   - 인증키 파라미터명
   - 페이지 번호 파라미터명
   - 페이지 크기 파라미터명
   - JSON/XML 응답 선택 파라미터
   - NCS분류 `기계` 필터의 실제 파라미터명. 코드값은 문서 기준 `R600015`
4. `.env`를 수정한다.

```text
ALIO_API_URL=명세의 호출 URL
ALIO_API_KEY=발급받은 인증키
ALIO_API_KEY_PARAM=명세의 인증키 파라미터명
ALIO_DEFAULT_PARAMS={"pageNo":"1","numOfRows":"100","type":"json"}
```

5. 설정을 점검한다. 이 명령은 인증키를 `***`로 가려서 출력한다.

```powershell
python -m job_radar check-config
```

6. 먼저 저장 없이 호출한다.

```powershell
python -m job_radar fetch-alio-public --dry-run --limit 5
python -m job_radar fetch-alio --dry-run --limit 5
```

`fetch-alio-public`은 공개 조회 endpoint를 사용하므로 `ALIO_API_URL`이 없어도 동작한다. `fetch-alio`는 발급키 기반 OpenAPI 호출용이며, 상세 명세의 실제 호출 URL이 필요하다.

7. 출력된 필드가 이상하면 실제 응답 샘플에서 개인정보/인증키를 제거한 뒤 `docs/alio_response_sample.redacted.json`로 저장하고, `alio_client.py`의 `FIELD_ALIASES` 또는 `ALIO_CODE_LABELS`를 보강한다.

8. 정상이라면 실제 저장을 실행한다.

```powershell
python -m job_radar fetch-alio-public --limit 5
python -m job_radar fetch-alio --limit 5
python -m job_radar list-jobs --min-score 1
python -m job_radar write-dashboard
```

## 완료 기준

1차 완료 기준:

- ALIO API 키를 넣고 수집 명령을 실행할 수 있다.
- 기계 분야 공공기관 채용공고가 `detected_jobs`에 저장된다.
- 같은 공고를 다시 수집해도 중복 저장되지 않는다.
- 점수와 매칭 키워드가 기록된다.
- 실패하면 `crawl_logs`에 원인이 남는다.

```text
init-db          SQLite 스키마 생성 및 키워드 seed

list-jobs        최근 감지 공고 출력
logs             최근 크롤링 로그 출력
write-dashboard  정적 HTML 대시보드 생성
serve            6시간 KST 자동 갱신 웹 서버 실행
```
