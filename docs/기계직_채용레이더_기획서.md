## 1. 제품 정의

### "채용 레이더"로서의 정체성

이 도구는 채용 서비스가 아니라 **개인용 자동 모니터링 시스템**이다.  
핵심 가치는 두 가지다: **발견 속도**와 **감시 범위의 확장성**.

> **3줄 정의:**  
> 내가 관심 있는 기업/직무의 채용공고가 올라오는 순간, 내가 먼저 알 수 있도록 하는 개인용 자동 모니터링 도구.  
> 공고를 읽어주거나 요약해주는 서비스가 아니라, 공고가 뜬 "사실 자체"를 빠르게 감지하고 알려주는 레이더.  
> 내가 직접 원문 링크를 확인하는 것을 전제로, 알림 → 링크 → 지원 결정까지의 흐름을 최대한 빠르게 만드는 도구.

### 필요한 기능 vs 필요 없는 기능

| 구분 | 기능 | 이유 |
|------|------|------|
| ✅ 필수 | 공고 감지 및 알림 | 핵심 목적 |
| ✅ 필수 | 감시 기업/소스 목록 관리 | 확장성의 기반 |
| ✅ 필수 | 크롤러 성공/실패 로그 | 감시 신뢰성 확인 |
| ✅ 필수 | 키워드 기반 필터/점수 | 노이즈 제거 |
| ✅ 필수 | 수동 크롤링 버튼 | 즉시 확인 필요 시 |
| ✅ 필수 | 중복 공고 감지 | 중복 알림 방지 |
| ⚠️ 선택 | 공고 상세 내용 저장 | 링크 클릭으로 충분히 대체 가능 |
| ⚠️ 선택 | 공고 만료 감지 | 있으면 편하지만 1차에는 불필요 |
| ⚠️ 선택 | 이메일/Discord 알림 | Telegram 이후 추가 가능 |
| ❌ 불필요 | 공고 자동 요약/AI 분석 | 복잡도 증가 대비 효과 낮음 |

| ❌ 불필요 | 이력서 관리 연동 | 이 도구의 스코프 외 |
| ❌ 불필요 | 합격/불합격 분석 | 목적이 다름 |

## 2. MVP 기능 범위

### 1차 버전 필수 기능

**크롤러 엔진**
- 타겟 URL 목록 관리 (DB 기반)
- HTML 파싱 — 공고 제목, 링크, 마감일, 회사명 추출
- 새 공고 감지 — unique_key (제목+기업+마감일 해시) 기준 DB 대조
- 스케줄러 — 3시간 주기 또는 1일 2회

**알림**
- Telegram 봇 연동 및 메시지 포맷 정의
- 점수 기준 알림 분기 (즉시 / 일반 / 무알림)
- 새 공고 감지 시 즉시 발송

**웹 대시보드 (최소 기능)**
- 오늘 감지된 공고 목록
- 기업별 마지막 크롤링 시간 + 상태
- 크롤러 성공/실패 표시
- 수동 전체 실행 버튼
- 간단한 로그 뷰어

**데이터 관리**
- 기업/감시 URL 추가·수정·삭제
- 키워드 규칙 관리

### 나중에 추가할 기능 (2차~)

- Discord / 이메일 알림
- 이메일 다이제스트 (1일 1회 요약)
- 공고 만료 감지 (URL 404 자동 확인)
- 공고 이력 분석 (기업별 채용 패턴 파악)
- 자동 기업 발굴 (플랫폼 키워드 검색 기반)
- 공고 상세 2차 크롤링 (선택적)
- 모바일 대시보드 반응형 개선

### 웹 대시보드 화면 목록 요약

| 화면 | 주요 내용 |
|------|-----------|
| 홈 (요약) | 오늘 감지 공고 수, 최우선 공고 5개, 크롤러 전체 상태, 수동 실행 버튼 |
| 공고 목록 | 날짜/산업군/기업/점수 필터, 원문링크 버튼 |
| 기업 관리 | 기업 CRUD, 소스 URL 관리, 기업별 수동 크롤링 버튼 |
| 크롤러 상태 | 소스별 최근 실행 결과, 실패 원인, 재시도 버튼 |
| 로그 | 실행 이력 타임라인, 실패 상세 메시지 |
| 키워드 & 규칙 | 긍정/부정 키워드, 점수 가중치 관리 |
| 알림 이력 | 채널별 발송 이력, 성공/실패 상태 |

## 3. 수집 대상 사이트 전략

### 공기업/공공기관

**1순위: 잡알리오 (job.alio.go.kr)**
- 공공기관 채용 공식 공개 창구로 대부분의 공공기관 커버 가능
- OpenAPI 제공 여부를 먼저 확인할 것 → 제공 시 API 방식 우선 활용
- 기관 유형·직종 필터를 활용해 기계/기술직만 모니터링 가능

**2순위: 각 기관 공식 채용 페이지**
- 잡알리오보다 빠르게 올라오는 경우가 있음
- 한수원, 발전 5사, 코레일, 공항공사 등 대형 기관은 직접 감시 추가 권장

**3순위: 나라일터 (gojobs.go.kr)**
- 공무원 채용 중심이지만 기능직/기술직 일부 포함

### 사기업

**원칙: 공식 채용 페이지 > 플랫폼**  
이유: 플랫폼 의존 시 API 차단 리스크가 있고, 플랫폼이 원본보다 늦는 경우가 있음.

**역할 분담**

| 소스 | 역할 | 특징 |
|------|------|------|
| 기업 공식 채용홈 | 1차 감시 대상 | 가장 빠름, 기업별 개별 크롤러 필요 |
| 그룹 통합 채용 포털 | 계열사 한 번에 감시 | 현대, LG, SK, 롯데 그룹 채용 포털 |
| 사람인 | 중견·중소기업 보완 | 공식 홈이 없거나 업데이트가 느린 회사 |
| 잡코리아 | 사람인 보완 | 일부 회사는 잡코리아에만 올리는 경우 있음 |
| 링커리어 | 신입/인턴 특화 | 신입 공고 조기 감지에 효과적 |
| 자소설닷컴 | 대기업 공채 시즌 집중 | 정기 공채 기간에 집중 모니터링 |
| 원티드 | 경력직 보완 | 스타트업·중견 일부 |
| 고용24 | 정부 지원 채용 | 잡알리오와 연계 |

**중복 공고 처리 전략**
- 공고 제목 + 기업명 + 마감일을 조합해 unique_key 생성 (MD5 해시)
- 이미 감지된 key면 DB update 없이 skip
- 알림은 최초 감지 1회만 발송
- 선택적으로: URL이 달라도 실질적으로 같은 공고인 경우를 위해 제목 유사도 검사 추가 (2차)

→ 이유: 채용 규모 자체가 크고 기계과 설비/공정 직무가 정기적으로 열림.

→ 이유: 배터리 시장 확대로 생산기술·설비기술 수요 최대. 공고 주기가 불규칙해 놓치기 쉬움.

현대자동차, 기아  
→ 이유: 국내 핵심 완성차, 생산기술·설비기술 채용 규모 큼.

**방산/항공우주**

→ 이유: 방산 확대 기조, 항공 MRO 수요, 기계직 핵심 직무 다양하고 경쟁력 높음.

- 산업군별 키워드 그룹핑

- 발송된 알림 목록

- 발송 시간, 대상 공고, 발송 상태 (성공/실패)

## 9. 데이터 모델

```sql
-- 기업 마스터
CREATE TABLE companies (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT    NOT NULL,
    type        TEXT,                           -- 'public' | 'private'
    industry    TEXT,                           -- 'auto' | 'semiconductor' | 'battery' | 'defense' | 'shipbuilding' | 'steel' | 'chemical' | 'robot' | 'public_inst'
    priority    INTEGER DEFAULT 2,              -- 1:최우선 / 2:일반 / 3:저우선
    homepage_url TEXT,
    is_active   INTEGER DEFAULT 1,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    memo        TEXT
);

-- 감시 소스 (기업별 크롤링 URL)
CREATE TABLE watch_sources (
    id                  INTEGER PRIMARY KEY AUTOINCREMENT,
    company_id          INTEGER REFERENCES companies(id),
    url                 TEXT    NOT NULL,
    source_type         TEXT,                   -- 'company_page' | 'platform' | 'api'
    platform            TEXT,                   -- 'jobinailio' | 'saramin' | 'jobkorea' | 'linkareer' | 'direct'
    crawl_frequency_min INTEGER DEFAULT 180,    -- 크롤링 주기 (분)
    selector_config     TEXT,                   -- JSON: CSS 셀렉터 설정
    last_crawled_at     DATETIME,
    status              TEXT DEFAULT 'active',  -- 'active' | 'error' | 'disabled'
    error_count         INTEGER DEFAULT 0,
    created_at          DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 감지된 공고
CREATE TABLE detected_jobs (
    id               INTEGER PRIMARY KEY AUTOINCREMENT,
    company_id       INTEGER REFERENCES companies(id),
    source_id        INTEGER REFERENCES watch_sources(id),
    title            TEXT    NOT NULL,
    url              TEXT,
    unique_key       TEXT    UNIQUE,            -- title+company+deadline MD5 해시
    posted_date      DATE,
    deadline         DATE,
    score            INTEGER DEFAULT 0,
    matched_keywords TEXT,                      -- JSON 배열: 매칭된 키워드 목록
    is_notified      INTEGER DEFAULT 0,
    is_expired       INTEGER DEFAULT 0,
    detected_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    raw_data         TEXT                       -- JSON: 원시 수집 데이터 (디버깅용)
);

-- 크롤링 로그
CREATE TABLE crawl_logs (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    source_id       INTEGER REFERENCES watch_sources(id),
    crawled_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    status          TEXT,                       -- 'success' | 'fail' | 'no_change'
    new_jobs_count  INTEGER DEFAULT 0,
    error_message   TEXT,
    duration_ms     INTEGER
);

-- 키워드 규칙
CREATE TABLE keyword_rules (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    keyword    TEXT    NOT NULL,
    category   TEXT,                            -- 'job_title' | 'industry' | 'exclude' | 'career_level'
    weight     INTEGER DEFAULT 0,               -- 양수: 가산 / 음수: 차감
    is_active  INTEGER DEFAULT 1,
    memo       TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 알림 이력
CREATE TABLE notifications (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    job_id          INTEGER REFERENCES detected_jobs(id),
    channel         TEXT,                       -- 'telegram' | 'discord' | 'email'
    sent_at         DATETIME,
    status          TEXT,                       -- 'sent' | 'failed'
    message_preview TEXT
);
```

**충분성 검토**

- companies + watch_sources 분리 → 기업 1개에 여러 소스 URL 연결 가능 ✅
- unique_key로 중복 공고 감지 → 중복 알림 방지 ✅
- score + matched_keywords 저장 → 점수 근거 추적·디버깅에 유용 ✅
- crawl_logs → 크롤러 신뢰성 모니터링 ✅
- notifications → 알림 재발송 방지 및 이력 확인 ✅
- 개인용으로는 이 6개 테이블로 충분. 이후 확장 시 `job_tags`, `crawl_schedules` 추가 가능.

## 11. 크롤링/감시 법적·윤리적 주의사항

**① robots.txt 준수**  
크롤링 전 대상 사이트의 `robots.txt`를 반드시 확인하고, 허용된 경로만 접근한다.  
예: `https://www.saramin.co.kr/robots.txt`

**② 요청 간격 제한**  
동일 도메인에 최소 2-3초 이상 딜레이를 유지한다. 연속 대량 요청은 절대 금지.  
크롤링 주기는 최소 1-3시간 간격으로 설정한다.

**③ User-Agent 설정**  
봇임을 위장하려 하기보다, 개인 프로젝트임을 명시하는 User-Agent를 사용하는 것이 윤리적이다.
```
User-Agent: JobRadar-Personal/1.0 (private use, non-commercial)
```

**④ 공식 API 우선 사용**  
잡알리오는 OpenAPI를 제공할 수 있으므로, 신청 후 API 방식을 우선 활용한다.  
공공데이터포털(data.go.kr)에서 고용 관련 API를 검색해 활용 여부를 먼저 확인한다.

**⑤ 로그인 우회 / 캡차 우회 금지**  
로그인이 필요한 페이지는 수집 대상에서 제외한다.  
캡차가 걸리면 해당 소스는 우회 시도 없이 포기하고 대안 소스를 사용한다.

**⑥ 개인정보 수집 금지**  
공고의 제목, 링크, 마감일 등 공개 정보만 수집한다. 지원자 정보, 연락처 등 개인정보는 절대 수집하지 않는다.

**⑦ 원문 링크 제공 원칙**  
공고 내용을 복사·재배포하지 않고, 반드시 원문 링크를 통해 사용자가 직접 확인하게 한다.

**⑧ 개인용 비공개 운영 유지**  
이 도구를 공개 서비스로 전환하거나 타인에게 제공하면 법적·윤리적 문제가 발생할 수 있다.  
철저히 개인용으로만 운영한다.

**⑨ 서버 부하 방지**  
공공기관 채용 사이트는 트래픽이 적으므로 과도한 요청이 서버에 영향을 줄 수 있다.  
최소 요청 원칙을 항상 유지한다.
